Winger: A Window Manager
A Comprehensive Guide

Last updated for v2.9.0 (17 Feb 2025)

View release notes

Winger helps you manage multiple browser windows, no matter how many. With Winger you can name them, find them, switch between them, move tabs between them, stash them away, and more.
Also: Slash (/) commands | Send to Window menu | Feedback/Support

First things first...

Your browser settings

In order for your windows and their names to persist across browser restarts: in Firefox Preferences, enable Open previous windows and tabs.

💡 Some basics/tips for working with tabs and windows

Select multiple tabs

Ctrl+click on each tab you want to select, one by one

Shift+click on a tab to select all the tabs between it and the currently focused tab, inclusive

Create a window

Ctrl+N to open a new window

Ctrl+Shift+P to open a new private window

Tear off tabs: With your pointer, drag-and-drop a tab off the tab bar to create a new window with it

Shift+click a link to open it in a new window

(Ctrl+click or middle-click opens it in a new tab)

Minimize a window

Click the minimize button (_, -, ▼, etc. depending on OS) at the top of a window

Windows and Linux: Alt+Space N

MacOS: Cmd+M

Windows: Win+▼Down to restore a maximized window or minimize a restored window

Close a window (Not the same as quitting the browser, which closes all windows)

Click the × button at the top of a window

Ctrl+Shift+W to close the current browser window

(Ctrl+W closes the selected tabs)

Common Windows and Linux shortcuts to close a window:
  • Alt+F4
  • Alt+Space C

The Winger panel

Click on the Winger button on your browser toolbar, or press F1, to open Winger's pop-up panel.

  • You can always check what the keyboard shortcut is by hovering your pointer over the Winger button.
  • If you don't see it on your browser toolbar, click on the 🧩 Extensions button, look for Winger, right-click it and choose Pin to Toolbar.
  • You can move browser toolbar buttons onto the tab bar – a reasonable spot for tab/window-related buttons! To be able to move buttons around, right-click the toolbar and click Customize Toolbar.

Every window is represented by a row, showing either its window title or name, and its tab count.

  • The window title (which mainly consists of the current tab title) is shown by default. If desired, it can be replaced with a given name: a unique, persistent identifier you can assign to a window.

The top row represents the currently focused window (current window).

Just below it is the panel omnibox.

If there are other windows, they are listed below the omnibox in this manner:

  1. open windows ordered from most to least recently focused
  2. followed by minimized open windows
  3. finally if stashing is enabled, "stashed windows"

At the bottom is the status bar showing the total number of open windows and tabs, and the help button.

You can interact with the panel by keyboard:

  • ▼Down, ▲Up, ▶︎Right and ◀︎Left to navigate the panel.
  • Enter to "click" a targeted item.

Basic actions

Switch to window

Click on a row, or press Enter on a targeted row, to switch to that window.

Send tab

The send button "sends" the selected tabs to the destination window without leaving the current window.

Ctrl+switch becomes a send. Thus, you can also Ctrl+click or Ctrl+Enter on a row to send tabs to that window.

Bring tab

The bring button switches to the destination window while "bringing" the selected tabs along. Essentially switch and send combined.

Shift+switch becomes a bring. Thus, you can also Shift+click or Shift+Enter on a row to bring tabs to that window.

Quick switch shortcuts

Keyboard shortcuts Alt+Q and Shift+Alt+Q let you switch to the next and previous non-minimized window, in alphabetical order (prioritizing named windows), without needing to open the panel.

Private windows

Winger can list, interact with, and create private windows if allowed private window access. Check your .

Tabs technically cannot move between normal and private windows, but Winger can perform a close equivalent: send/bring closes the selected tabs in the current window and reopens them in the destination window.

Send/bring buttons that will reopen tabs appear purple when targeted or hovered over.

Reopened tabs

Reopened tabs are the result of sending/bringing tabs between normal and private windows, and also the result of unstashing.

  • Reopened tabs start out in an unloaded state, if possible.

    • Reopening multiple tabs in an unloaded state ensures that such actions remain non-intensive and instantaneous, no matter how many tabs. Otherwise, loading large numbers of tabs all at once may briefly strain your system, potentially slowing your browser to a crawl.

  • Reopened tabs with privileged URLs that Winger cannot open ("about:" pages, local files, etc) become temporary placeholder tabs. Placeholder tabs will inform you of a manual step required to properly restore them. (Example)

  • Reopened tabs preserve any pinned and reader-view tab states, but not most other tab/page states such as tab history, scroll position and unsubmitted form data. Container states disappear in private windows.

Name windows / Edit mode

To enter edit mode, click on the current window's name, or press ▲Up Enter from the omnibox.

  • Alternatively, type /edit Enter in the omnibox to toggle edit mode on or off.

In edit mode, all names become targetable and editable, and basic actions disabled.

Names should be unique and may not start with a forward slash (/).

A valid window name is saved when you de-target it (by clicking or navigating away), or when you press Enter in a name field. Invalid names, such as duplicates, are reverted upon any save attempts.

Pressing Enter in a name field exits edit mode.

Closing the panel, by clicking outside it or by pressing Esc, exits edit mode without saving changes to the targeted name.

Where do names appear?

Apart from the Winger panel, window names appear in:

  • the title preface (if enabled in ), added to the window title shown in the title bar. The title preface includes customisable decorators: the prefix and postfix. e.g.
    learn -  ...
    [learn]  ...
  • Window titles in turn show up in:
    • your operating system's task manager/switcher, for example while Alt+Tab-ing.
    • your browser's Window menu, on some operating systems e.g. MacOS.
  • the badge (if enabled in ) on the Winger button , showing the first few characters.
  • the tooltip that appears when you hover your pointer over the Winger button .
  • Send to Window tab and link context menus.

💡 Naming tips

  • Use shared keywords in names to help categorise related windows, e.g.
    learn
    learn: SQL
    learn: guitar
    Later this can help you only list a particular category.
  • Similarly, you can use symbols, e.g. "!" to indicate high priority:
    jobhunt !
  • 🔥 You can use emoji in names and decorators. Windows and MacOS have built-in emoji pickers (Win+. and Cmd+Ctrl+Space respectively), or you can install an emoji picker extension.
    • But best use emoji as visual flair, not in place of actual words, because you want easily typeable, findable names.
  • If you use multiple Firefox profiles, you can have different title preface decorators as a way to distinguish between windows of different profiles. e.g.
    Work {learn: SQL}  ...
    🏡 learn: guitar ~  ...
  • Since the button badge can only show a few letters, offer some ways to improve its usefulness:
    • Move all emoji in a badge to the start, if you find them helpful for identifing a window.
    • Transform badge contents with a regular expression (regex) that matches a smaller portion of it.
      e.g. [^:]+$ to show only everything after the last ":" if any.
      e.g. \b\w with global enabled, to show only the first letter of each word.

The omnibox

The omnibox is a multi-purpose, multi-talented input field that vastly accelerates operating Winger by keyboard. Here's what you can do with it:

Perform basic actions on the topmost other window

Press Enter, Ctrl+Enter or Shift+Enter in the omnibox to switch, send or bring to the first window listed below it.

This means upon opening the panel, you can via keyboard immediately perform any of the basic actions on the previously seen window.

Find/filter windows

Looking for a window? Type its name or title, or part of it.

This filters the list of other windows. Thus you can change which window appears right under the omnibox. Also, a shortened window list can be traversed with arrow keys in fewer steps.

Filtering works in edit mode too.

Combine these two abilities to quickly switch to another window by keyboard, like so:

  1. F1 to open the panel.
  2. If the desired window is not immediately below the omnibox, start typing its name or title until it is.
  3. Enter.

Slash commands

Certain actions can be performed by typing one of these in the omnibox followed by Enter.
Slash commands auto-complete as you type, and work if partially typed too.

/help or /h or /
Open this help page.
/settings or /s or
/options or /o
Open .
/edit or /e
Toggle edit mode.
/name or /na
Name the current window without entering edit mode, with a name following the command e.g. /name socials.
Note: if the name is ommitted, the current window's name will be cleared.
/stash or /st
Stash the current window.
Shift+Enter to prevent the window from closing.
Available if stashing is enabled.
/viewstash or /v
Toggle listing "stashed windows".
Available if stashing is enabled.

Advanced commands
/extractname or /ex
Extract a name from the current window's title matching a regular expression (regex), and give the window this name. A regex pattern must follow the command. If it contains a capture group ((...)), only the captured result is used.
Useful for migrating names from another window-naming extension that uses title prefaces.
e.g. To extract "Home" from "Home - How to DIY": /extractname .*?(?= - ), or with capture group, /extractname (.*?) - .
e.g. To extract "july trip" from "[july trip] Search restaurants": /extractname (?<=\[).*?(?=\]), or with capture group, /extractname \[(.*?)\].
/extractallnames or /exa
Like /extractname but applies the given regex to all windows at once. Use with caution!
/new or /n
Open a new window; private if current window is private.
Name it as well if a name follows the command e.g. /new Eng homework.
Shift+Enter or Ctrl+Enter to invoke /pop or /kick respectively.
/pop or /p
("Pop" tabs out of the current window)
Bring selected tabs to a new window; private if current window is private. Equivalent to dragging selected tabs off the tab bar.
Name the window as well if a name follows the command e.g. /pop News.
/kick or /k
("Kick" tabs out of the current window)
Send selected tabs to a new window while staying on the current window; private if current window is private.
Name the window as well if a name follows the command e.g. /kick read later.
/newprivate or /np
/newnormal or /nn
/popprivate or /pp
/popnormal or /pn
/kickprivate or /kp
/kicknormal or /kn
Same as /new, /pop and /kick but explicitly specifying private or normal new window.
Available if Winger is allowed private window access.

Send-to-Window menu command

Tab context menu

Right-click on a tab to get Send to Window in the context menu, which shows a submenu listing all other windows. Then click on a window name/title to send the tab there. (Shift+click to bring instead.) This works whether or not the tab was selected.

If you right-clicked on one of multiple selected tabs, the action will apply to all of them, as you would expect.

Link context menu

Right-click on a link in a page to get the same Send to Window menu command. Then click on a window name/title to open the link there. (Shift+click will also switch to the window.)

💡 What are those underlined letters?

Some operating systems like Windows and Linux provide menu access keys, for accessibility and rapid interaction.

In an open menu (and a browser menu bar activated with Alt), an underlined letter indicates you can tap that key to access ("click") the menu item. (Shift+key is the Shift+click equivalent.)

A menu item with no access key can be accessed with its first letter, if it's not already an access key for another menu item.

If a menu has multiple items sharing the same access key, it's slightly different: you may need to tap the key more than once to highlight the desired item, then confirm with Enter (or Shift+Enter).

Stash windows

If you want to close a window to reduce clutter, but also be able to restore it at any time, you can stash it. Stashed windows are stored as bookmark folders, which you can then unstash.

Certain window/tab properties – such as pinned tabs, muted tabs, and more importantly, privacy-related ones such as private windows and containers – are also preserved in the stash, allowing a secure, near-complete restoration. (Tab groups are currently not supported)

Enable stashing in to start using it.

Stashing

To stash a window: in the panel, click on the window's stash button. Or type /stash Enter in the omnibox to stash the current window.
The window will close and its tabs saved as bookmarks, in a new bookmark folder named after it, located in the stash home.

  • For each tab, if any of the following properties are found, they are captured as a JSON-format property annotation in the new bookmark's title: pinned, muted (audio off), active (currently focused), parent-child relationship (tab opened from another tab), and container name.
  • A private window's private state is likewise captured in the new stash folder's title.
  • If a name-matching bookmarkless folder is found in the stash home, Winger will use it instead of creating a new folder.

The send action available to bookmark folders allows you to stash the currently selected tab(s) into them.
The tab(s) will be closed and saved as bookmark(s) in the folder.

Accessing

Look in the stash home folder to find your stash folders ("stashed windows") via the usual ways to view bookmarks. Right-click any folder or bookmark for menu commands Winger → Send Tab Here and Winger → Unstash (available even outside the stash home).

To display stash home contents in the panel, type /viewstash Enter in the omnibox.

  • Once displayed, "stashed windows" are findable, renamable, can be sent tabs to, and can be unstashed. (No switching or bringing tabs to them since they're not real windows.)

Unstashing

To unstash a bookmark folder: right-click on it and click the Winger → Unstash menu command. Or in the panel, click on its unstash button.
The folder will be deleted, and an identically-named window will open and populate with the formerly-bookmarked tabs, reproduced with their properties as annotated, if any.

  • The menu command is also available when right-clicking on a bookmark; it will delete and open the bookmark as a new tab in the current window.
  • Unstashed tabs are reopened tabs: unloaded if possible, as placeholder tabs if necessary.
  • When unstashing a contained tab, Winger assigns a container that matches the captured container name. If no match is found, Winger will create a new container with that name, guaranteeing containment.
  • If the folder has subfolders, nothing happens to them and the folder itself is not deleted; it is only emptied of its own bookmarks.

Stash-copy / Unstash-copy

Hold down Shift while performing a stash/unstash action to make it non-destructive: windows/tabs being stashed are not closed, folders/bookmarks being unstashed are not deleted. This effectively lets you create "copies" of windows and tabs.

Stash home location

The stash home folder can be one of the three root folders – Bookmarks Toolbar, Bookmarks Menu, Other Bookmarks – or a user-named folder inside a root folder. You can specify it in .

Changing the stash home location will not move existing stash folders.

If the stash home is a root folder, its last separator is considered the stash separator, partitioning stash folders (to its right) from other bookmark items, if any (to its left). This allows you to have statically-positioned, non-stash items coexisting with stash folders. Winger will auto-add a separator if necessary.

💡 Stash tips

  • As a "format" for stashed windows, bookmarks are simple and transparent; they are accessible and useful independently from Winger – to you, to Firefox Sync, to other extensions, etc. – and you don't lose them if you uninstall Winger. (Note however, currently only Winger can process property annotations.)

  • With Firefox's bookmark syncing, you can stash windows in one profile/computer and unstash them on another – effectively moving browser windows between profiles/computers!

  • You are free to manipulate stashes as regular bookmarks and folders: add, edit, delete, reorder, and so on.

    • You can move folders in and out of the stash home to include/exclude them in/from the panel.

    • Winger ignores invalid or incorrect annotations. So you can move a "containered tab" bookmark into a "private window" folder, an "active tab" into a "window" that already has one, etc. – nothing surprising can happen. Also, you can manually add/edit/remove annotations with little risk.

    • If you deleted a bookmark or folder by accident, no worries: Go to Manage Bookmarks (via browser's Bookmarks menu or via shortcut Ctrl+Shift+O), and press Ctrl+Z to undo the last bookmark action.

💡 Tips for managing lots of tabs and windows

  • Spawn as many windows as you need for more granular tab organisation!

    • Divide and conquer: split up a highly-populated window into several windows, so you can focus on one set of tabs at a time. Hopefully this helps you work through them easier and close them sooner.

  • If you are hunting for a browser window among several, don't bother Alt+Tab-ing endlessly. Just type in the omnibox the name of the window you want.

  • Minimize windows to "set aside for later" – Winger lists them last on the panel, out of your way so to speak.

    • Wish you could "pin" windows on the panel so they stay near the top? Just minimize everything (Windows/Linux Win+M; MacOS Cmd+Opt+M) and then restore the few windows you want to focus on. Voilà.

  • If you are not ready to permanently close a window you hardly use, consider stashing it.

  • Key to the freedom to have lots of tabs open is unloading any unused ones, which frees up memory. Firefox does this automatically; its tab unloading queue can be seen on the about:unloads page.

    • Tab unload (a.k.a. discard, suspend) extensions are a popular way to wield this ability with greater control, for example if you prefer a more aggressive treatment than Firefox's (recommended with lots of tabs), or if you want to manually unload selected tabs, and so on.

    • Winger can provide a simple treatment: unload tabs in minimized windows, including any tabs sent to them. If this suits you, enable it in .

    • You can have too many windows, even with tab unloading, because every window has a current tab that cannot be unloaded. A way around this problem is to have a blank/new/home tab as the current one for as many of your idle windows as possible.

  • Winger does not provide tab search. Here are ways to find tabs:

    • In the address bar, type % Space followed by your search terms.

    • Firefox View. Access it via its toolbar button, or the menu bar Tools → Firefox View, or the url about:firefoxview.

    • Extensions like TabSearch and Tabhunter.
