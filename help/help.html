Winger: A Window Manager

Quick Guide

Last updated v1.3.1

The Winger Panel

Click on the Winger button on your browser toolbar, or press , to open the pop-up panel.

Tip: You can move browser toolbar buttons onto the tab strip – a reasonable spot for tab/window-related actions! To move buttons around, right-click the toolbar and click Customize to enter Customize Firefox mode.

Every window is represented by a row, showing its name and tab count.

The topmost row represents the currently focused window (current window).

Below it is the panel omnibox.

If there are other windows, they are listed below the omnibox; the first row representing the most recently focused (last seen) window, followed by the next most recently focused window, and so on.

Below the list is the status bar showing the total number of tabs and windows open.

Lastly at the bottom is the panel toolbar.

Common Actions

• Switch Window

Click on a row to switch (change focus) to the corresponding window.

• Send Tab

The send button "sends" the selected tab(s) to the corresponding window without leaving the current window.

Ctrl+switch becomes a send. Thus, you can also ctrl+click a row to send tabs to that window.

• Bring Tab

The bring button switches to the corresponding window while "bringing" the selected tab(s) along. Essentially switch and send combined.

Shift+switch becomes a bring. Thus, you can also shift+click a row to bring tabs to that window.

Private windows

Winger can list and interact with private windows if it is allowed private window access. Check .

Private windows are marked with purple names.

Tabs technically cannot be moved between normal and private windows, but Winger can perform a close equivalent: send and bring will close tabs in the current window and reopen them in the destination window. Buttons that will perform this are shown purple.

  • This will not work for certain types of tabs, such as file:// pages and some about: pages.

  • Pinned, discarded (a.k.a. suspended) and reader-view tab states will be preserved.

  • Other tab and page states – including tab history, scroll position and unsubmitted form data – will be lost in the process.

Pinned tabs

Send and bring can work on pinned tabs if enabled in .

Name Windows

Winger assigns every window a default name (Window 1, Window 2 and so on), visible when a window has no user-given name.
Unlike given names, default names are temporary and not guaranteed to persist across browser sessions.

The edit button lets you name/rename a window.

You can also edit the current window's name by double-clicking it. NEW

When the panel is in Edit Mode, you can type in a new name and then use the following keys:

  • Enter to save the currently edited name and exit Edit Mode.

  • Up or Down to save the currently edited name and edit another window name.

  • Esc to close the panel, exiting Edit Mode without saving any changes to the currently edited name.

Names must be unique and may not begin with a forward slash ("/").

Naming tips

  • In Edit Mode, the status bar will show the title of the targeted window's active tab, to help you identify any windows that are not visible. NEW

  • You can include emojis in names. Windows and Mac have built-in emoji pickers (Win+. and Cmd+Ctrl+Space respectively). There are also emoji extensions.

Where a window name appears

  • Title bar. Contents of the title bar also show up in your operating system's task manager or switcher (e.g. while Alt-Tabbing), even if you've set Firefox's title bars to be hidden.

  • Tooltip that appears when you hover your pointer over the Winger button . NEW

  • Button badge on the Winger button (showing the first few characters of a name), if enabled in . NEW

  • And of course, open the panel any time to see the current window's name in bold.

Keyboard Navigation
& The Omnibox

Press to open the panel. You can always check what the keyboard shortcut is by hovering your pointer over the Winger button .

You can use the arrow keys to navigate the panel: Up and Down to select rows, Left and Right to select buttons. Press Enter or Space to activate a row or button.

You should also use Winger's omnibox – a multi-purpose, multi-talented input field – to vastly accelerate operating Winger by keyboard.

Primary omnibox powers

  • Start typing in a window name to filter the list of other windows.

  • Press Enter to switch to the first window listed below the omnibox.

    Recall that Ctrl+switch becomes a send and Shift+switch becomes a bring.

    Tip: Pressing either of these modifier keys while the omnibox is empty will hint what it does.

Thus in summary, to swiftly switch OR send tabs OR bring tabs to a window via keyboard:

  1. to open the panel.

  2. If the desired window row is not already immediately below the omnibox, start typing its name until it is.

  3. Enter OR Ctrl+Enter OR Shift+Enter.

    Alternatively, use arrow keys to select the desired row or button.

Slash commands

Certain actions can be performed by typing one of these slash commands in the omnibox, followed by Enter. Slash commands auto-complete as you type.

/help or /h or just /
Open this help page
/edit or /e
Enter Edit Mode to rename windows
/settings or /s
Open Winger settings

Tip: Customise the panel

Once you get comfortable enough with the omnibox, you have the option of removing any buttons that you no longer use.

Remove them all if you like a minimalist panel!

Context Menus

Winger commands are available in the following context menus once there are at least two windows open.

Tab context menu

Right-click on a tab to get the Send Tab to Window‚Ä¶ menu command, which opens a submenu of other windows (sorted by recently focused).

Click a submenu item to send the tab to the chosen window. (As you might expect, Shift+click to bring the tab instead.)

If there are multiple tabs selected and one of them was right-clicked, the command will apply to all of them.

You can also right-click on an unselected tab to use this command on it.

Link context menu

Right-click on a link in a page to get the Open Link in Window‚Ä¶ menu command, which opens the same submenu as above.

Click a submenu item to open the link in the chosen window. (Shift+click will also switch to the window.)

Tips for Managing Large Tab/Window Counts

  • Use a tab discarding/suspending extension to minimise memory usage on less-visited tabs and windows.

  • Divide and conquer with "window mitosis": split up a highly-populated window into several windows so you can focus on one set of tabs at a time. Hopefully this helps you work through them easier, and close them sooner.

  • If you like to proactively keep the number of windows under a certain number, you can always check the window count on the panel status bar. Another cue you can use to start closing windows is when the list of other windows to overflows and scrolls, which happens at about 12 windows.

  • Bookmark tabs and windows that you probably won't visit for a while, and close them.

    • Ctrl+D to bookmark the current tab, or Ctrl+Shift+D to bookmark all tabs in a window. (You can also find these commands in the Bookmarks menu (Alt B).)

    • A future Winger update aims to streamline this. Stay tuned...

    • Try some tab saving extensions for alternatives to regular bookmarking.

  • The "List all tabs" button, which opens a menu of every tab in the current window, appears whenever the tab strip overflows. Its shortcut is Ctrl+Shift+Tab.

    • Each menu item can be right-clicked, just like a tab, to open its context menu. This is as close as Firefox currently gets to native vertical tabs!
